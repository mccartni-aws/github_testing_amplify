Q: What are you using to serve the front-end?
A: All of our static web content including HTML, CSS and JavaScript, images and other files are managed by Amplify. You, the end user, then access this site using the public website URL exposed by the Amplify console. No web servers are required to make our site available.
Q: How did you ensure CI and CD?
A: Amplify provides a git-based workflow for continuous deployment and hosting. A push to the Git repi would then udate the application in real-time.
Q: How did you provide dynamic functionality on certain web pageto the user?
A: JavaScript executed in the browser sends data from a front-end API to a back-end lambda function through API gateway.
Q: How did you ensure secure sign-up and sign-on?
A: We use Amazon Cognito to provider user management and authentication functions to ensure secure sign-up and sign-on. After users submit their registration, Amazon Cognito will send a confirmation email with a verification code to the address they provided. To confirm their account, users will return to your site and enter their email address and the verification code they received. When users sign in, they enter their username (or email) and password. A JavaScript function then communicates with Amazon Cognito, authenticates using the Secure Remote Password protocol (SRP), and receives back a set of JSON Web Tokens (JWT). The JWTs contain claims about the identity of the user and will be used in the next module to authenticate against the RESTful API you build with Amazon API Gateway.
